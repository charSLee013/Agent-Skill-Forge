# Ultra Workflow

Read this file after the user confirms Ultra. Ultra targets comprehensive domain coverage, expert judgment, high-density teaching, publication-grade frontend work, and independent review.

## Shared Brief

The primary Agent creates one shared brief from the Course Contract. Include:

- learner, prerequisite gap, target capability, and target use;
- domain scope, main trunk, complete topic ambition, and graduation artifact;
- source and evidence expectations;
- static delivery target and page topology;
- visual, information-density, formula, and interaction expectations;
- tracer contents and four review gates.

Every specialist works from this shared brief. The primary Agent owns synthesis and final decisions.

## Temporary Workspace

Use the session working directory declared by `SKILL.md`, with the Ultra profile name:

```text
${TMPDIR:-/tmp}/teach-ultra-{unique-id}/
  research/
  drafts/
  reviews/
  screenshots/
  tests/
```

Place raw research, intermediate drafts, subagent feedback, screenshots, browser profiles, logs, and test fixtures in this directory. Place approved learner-facing HTML, shared assets, source notes, capstone materials, and final review results in the course directory.

Clean the owned temporary directory after the final publication passes every gate.

## Phase 1: Parallel Discovery

Dispatch specialists according to the domain:

- **Domain Expert**: define authentic expert tasks, standards, decisions, and failure diagnoses.
- **Evidence Researcher**: collect sources, facts, data, formulas, standards, and provenance.
- **Learning Architect**: map prerequisites, progression, complete topic coverage, and transfer.
- **Mechanism Auditor**: reconstruct formulas, processes, state changes, boundaries, and examples.
- **Assessment Designer**: define practice, graduated feedback, capstone, and rubric.
- **Visual and Frontend Director**: define the page system, information hierarchy, technical representations, interaction grammar, responsive behavior, and static enhancement model.

Each specialist writes findings to the temporary workspace. The primary Agent checks coverage, resolves contradictions, and produces one integrated course architecture.

Complete this phase when every Course Contract field has evidence-backed content and every prerequisite gap has a planned bridge.

## Phase 2: Integrated Architecture

The primary Agent synthesizes:

- a memorable main trunk;
- prerequisite and concept dependency maps;
- Core Path, Deep Dives, Reference Atlas, Evidence Map, and Expert Branches;
- page responsibility and content-object assignments;
- claim-to-source and formula verification maps;
- practice progression, capstone, and feedback system;
- one subject-appropriate visual and frontend system;
- mechanism representations and interaction opportunities;
- the static publication topology.

Complete this phase when the architecture fully specifies every page's structural decisions.

## Phase 3: Tracer Set

Build one integrated tracer set:

- course map;
- the densest representative lesson;
- one reference or deep-dive page;
- one complete practice or lab state when interaction serves the Course Contract.

Choose a lesson that exercises the hardest combination of long-form reading, formulas or process detail, evidence, tables, diagrams, practice, responsive layout, ordinary navigation, and progressive enhancement.

Run all four review gates. Send raw tracer artifacts and neutral gate prompts to fresh reviewers. Reviewers write findings in the temporary workspace. The primary Agent applies concrete repairs and reruns affected gates.

Present the passing tracer to the user with the Course Contract and one recommended interpretation. Expand after the user's single calibration.

## Phase 4: Course Expansion

Delegate coherent learning arcs with explicit page responsibilities. Give each contributor the shared brief, accepted tracer, vocabulary, evidence map, visual system, and navigation contract.

The primary Agent integrates each arc, checks cross-page terminology and prerequisite continuity, and preserves one publication topology. Evidence and design specialists review representative additions during expansion.

Complete this phase when every declared page, source note, activity, content object, and capstone artifact exists in the final output tree.

## Phase 5: Final Review

Use fresh reviewers for:

1. Publication
2. Learning and Fidelity
3. Design
4. Enhancement

Give reviewers final learner-facing artifacts and the minimum Course Contract context needed for the assigned gate. Keep expected conclusions and prior diagnoses outside their prompts.

The primary Agent repairs findings in severity order, reruns affected checks, records final results, verifies the declared static target, and cleans the owned temporary workspace.

## Runtime Adaptation

Use parallel specialists when the environment supports subagents. Run the same roles sequentially when it provides one Agent context. Record the available review independence in the final result while preserving the Ultra quality bar.
