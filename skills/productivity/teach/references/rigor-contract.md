# Rigor Contract

Use a rigor contract when a course must be more than a casual explanation.

## Profiles

| Profile | Required behavior |
|---|---|
| `lightweight` | Explain the point clearly and add retrieval prompts. |
| `guided` | Add sequence, examples, practice, and feedback. |
| `evidence-intensive` | Make major claims visibly justified and keep detailed anchors in the source matrix. |
| `report-grade` | Use a structured object map, explicit boundaries, and mechanism explanations. |
| `expert-audited` | Add skeptical expert review and block shallow or imprecise artifacts. |

## Contract Fields

Before writing student pages for `evidence-intensive` or stricter profiles, record:

- Goal.
- Reader level.
- Target use.
- Genre: tutorial, field guide, report, paper course, implementation manual, reference manual, or interactive explainer.
- Rigor profile.
- Evidence style: inline evidence boxes, collapsed appendix, source matrix, or mixed.
- Success criteria.
- Out of scope.

Write one explicit sentence:

`This course will be written as a {genre}, with {evidence style}, for {reader level}.`

## Completeness Rule

The smallest complete artifact is the smallest unit that satisfies the rigor contract, not the shortest page.

For a strict course, one complete phase may be a single deeply evidenced chapter with one mechanism explanation and one audit. It is not complete if it spreads shallow coverage across many pages.

## Tone

For report-grade or expert-audited work, use objective scope language.

Prefer:

- "This artifact is scoped as..."
- "The implementation provides..."
- "The evidence supports..."
- "The available public evidence does not establish..."
- "This section treats X as Y because..."

Avoid:

- "Obviously..."
- "Just..."
- "Do not get trapped..."
- Repeated negative framing where a positive scope definition would be clearer.
