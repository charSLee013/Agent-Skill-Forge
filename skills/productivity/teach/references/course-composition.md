# Course Composition

Compose a course from the subject's structure and the learner's goal. The course map shows the chosen route; each lesson carries one meaningful part of that route.

## Composition Inputs

Record:

- learner and starting point;
- desired capability;
- subject objects and vocabulary;
- available sources and examples;
- useful practice and transfer context;
- the next action that makes learning observable.

## Module Library

Choose the modules that fit:

| module | purpose | common output |
|---|---|---|
| orientation | establish purpose, vocabulary, and main line | course map or opening lesson |
| foundation | establish prerequisites and representations | lesson or reference |
| mechanism | explain objects, variables, states, steps, and constraints | lesson, trace, table, or visual spec |
| evidence | connect claims to sources and examples | lesson plus source notes |
| practice | make the learner retrieve, apply, and receive feedback | lesson or exercise |
| transfer | connect the idea to a concrete use | reference or closing lesson |
| synthesis | compare, build, explain, or review the whole subject | final lesson or reference |

Use any order that follows the learner's dependency path. A course map can revisit a module when later evidence sharpens an earlier explanation.

## Structured Objects

For repositories, systems, protocols, papers, workflows, and tools, map the dimensions that matter:

- purpose;
- interfaces;
- representations;
- processes;
- states;
- constraints;
- environment;
- verification;
- open questions.

Use `templates/structured-object-map.md` as a planning surface. Keep paths, symbols, and detailed anchors in supporting Markdown.

## Mechanisms

For a non-trivial mechanism, explain the object, inputs, variables, initial state, one step, whole process, constraints, and failure modes before showing code, equations, or diagrams. Use `templates/mechanism-visual-spec.md` when a visual or interaction will improve comparison.

## Page Composition

Each lesson can combine:

- a main-line summary;
- a memory target;
- a plain-language explanation;
- a source or example;
- a retrieval prompt;
- a next action.

Keep the page focused on the learner's next useful understanding. Place lookup tables and detailed source notes in reference pages or supporting Markdown.
