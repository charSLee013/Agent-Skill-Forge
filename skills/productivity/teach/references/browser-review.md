# Browser Review

Use Chrome DevTools as the primary review surface for HTML courses when the browser tool is available. The review checks the rendered artifact that the learner will actually use.

## Review Pass

Open the course map and follow every local course link. Inspect at least one representative lesson and reference page at:

- a desktop viewport;
- a narrow mobile viewport;
- every meaningful interactive or visual state.

Inspect:

- page title, heading hierarchy, main line, and next action;
- Console errors and warnings;
- Network failures and missing assets;
- local links and navigation;
- text flow, clipping, overflow, and overlap;
- table, diagram, image, canvas, and interaction readability;
- keyboard focus and visible state changes;
- the text explanation that accompanies a visual mechanism.

Capture screenshots for important states and record the result in the session or `review.md`.

## Content Pass

Read the HTML and supporting Markdown as a learner. Confirm that the course makes its purpose, current idea, source support, scope, practice, and next action easy to find.

When Chrome is unavailable, record the environment limitation and complete the content review. Browser review remains the preferred evidence for rendered behavior.
