# Agent Skill Forge Maintenance Guide

This is a neutral personal `Agent-Skill-Forge` repository. It is not an upstream mirror.

This file is the repository's only Agent instruction source. Do not create parallel or nested instruction files.

## Structure

Skills are organized into three shipped buckets under `skills/`:

- `engineering/` - code work, planning, local issue workflow, architecture, implementation, debugging, and testing.
- `productivity/` - non-code planning, grilling, handoff, Teach v2, and skill-writing guidance.
- `research/` - research retrieval and document-building workflows.

Every shipped skill must have:

- a `SKILL.md`
- an entry in the matching bucket `README.md`
- an entry in the top-level `README.md`
- an entry in `.claude-plugin/plugin.json`

Each skill entry in a README must link the skill name to its `SKILL.md`.

## Invocation

Every `SKILL.md` is either:

- user-invoked: `disable-model-invocation: true`, reachable only when the human asks for it
- model-invoked: model- or user-reachable through its description

For the full definitions, description conventions, and why a user-invoked skill can invoke model-invoked skills but never another user-invoked one, see [docs/invocation.md](./docs/invocation.md).

## Local Workspaces

Engineering skills use a repo-local `.codex/agents/` workspace for PRDs, issues, and triage notes. That workspace is generated by `/setup-agent-skills` and should stay private to the consuming project.

This repository itself should not keep long-horizon run artifacts, release-process artifacts, or upstream git history.

## Validation

Run these checks after changing a skill, installer, or repository contract:

```bash
./scripts/list-skills.sh
bash scripts/test-engineering-contracts.sh
bash scripts/test-install-shape.sh
git diff --check
```
